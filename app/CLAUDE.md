# Project Instructions

## Project Overview

This repository contains the BREATH//BREAK application.

The app is a minimalist breathing practice tool inspired by Tonglen practice.  
Users perform short emotional exposure breathing sessions and track their practice history.

The product philosophy:

- Minimal UI
- Fast sessions
- Emotion-focused breathing
- No unnecessary friction

## Tech Stack

- Next.js
- React
- Tailwind CSS
- Supabase (planned / partial)
- TypeScript if already present

## Working Rules

Before editing files:

1. Read the repository structure first.
2. Explain the planned changes before making them.
3. Prefer **small focused changes** instead of large rewrites.

## Code Style

- Follow existing patterns in the repository.
- Do not introduce new libraries unless necessary.
- Keep components simple and readable.
- Avoid unnecessary abstractions.

## UI Rules

- Preserve the existing visual style.
- Do not redesign UI unless explicitly requested.

## Safety Rules

Never:

- rewrite large sections of the app without explanation
- introduce heavy dependencies
- break existing features

## When implementing changes

Always provide:

1. A short implementation plan
2. Files that will change
3. A summary of what was modified

## Pull Request Summary Format

When suggesting a PR, include:

- Summary
- Files changed
- Why the change improves the system
